Мои навыки

Основное, что определяет специалиста - это его знания и умения. И в первую очередь программист должен владеть языками, на которых будет писать код.

Гибкие навыки

Умение находить информацию
Программист не может знать всё. Но хороший программист может разобраться почти во всём - только предоставьте ему доступ в интернет.
Чтение чужого кода
Никто не пишет большие проекты в одиночку. Понимать, что делает твой товарищ - важное качество.
Работа в команде
Команда должна дискутировать и приходить к лучшему решению. Члены команды должны уметь это делать вежливо и конструктивно.

Галерея достижений

В прошлом году я работала над приложением, которое помогало выбрать пользователю наиболее подходящий под его запросы ресторан. Одним из небольших кусочков приложения был расчёт расстояния в метрах от местоположения пользователя до точки назначения. Я нахожу его довольно красивым и интересным, так как тут пришлось учитывать то, что расчёт ведется по поверхности Земли, а не в двумерной системе координат.

    //The earth's radius, in meters.
    private static final double EARTH_RADIUS = 6371009;

    private static double hav(double x) {
        double sinHalf = sin(x * 0.5);
        return sinHalf * sinHalf;
    }

    private static double arcHav(double x) { // арк.гаверсинус
        return 2 * asin(sqrt(x));
    }

    public static double getDistance(Point userPoint, Point restaurantPoint) {
        double xFirstRad = toRadians(userPoint.getLatitude());
        double yFirstRad = toRadians(userPoint.getLongitude());
        double xSecondRad = toRadians(restaurantPoint.getLatitude());
        double ySecondRad = toRadians(restaurantPoint.getLongitude());

        double havDistance = hav(xFirstRad - xSecondRad) +
                hav(yFirstRad - ySecondRad) * cos(xFirstRad) * cos(xSecondRad);
        double distanceRadians = arcHav(havDistance);

        return distanceRadians * EARTH_RADIUS; // in meters
    }
